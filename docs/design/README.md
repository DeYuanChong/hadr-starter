# Design archive

Background documents from the project's planning phase, kept for context. They
explain *why* the system is shaped the way it is. None of them is required to
run, deploy, or contribute to the project.

| Document | What it is |
| --- | --- |
| [`prd.html`](./prd.html) | The product requirements document — the full specification the implementation was built against. |
| [`requirements.md`](./requirements.md) | Initial idea capture: the raw requirements that preceded the PRD. |
| [`grilling-questions.md`](./grilling-questions.md) | Working log of open questions raised against the requirements and how each was resolved. |
| [`implementation-notes.md`](./implementation-notes.md) | Running log of decisions and deviations kept during implementation. |

For the **authoritative, current** design record, see:

- [`../adr/`](../adr/) — Architecture Decision Records, one per significant choice.
- [`../../CONTEXT.md`](../../CONTEXT.md) — the domain vocabulary (Story, Triage severity, Story state, …).
- [`../../ARCHITECTURE.md`](../../ARCHITECTURE.md) — the system architecture overview.
